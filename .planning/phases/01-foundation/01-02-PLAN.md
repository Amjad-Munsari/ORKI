```typescript
import {getRequestConfig} from 'next-intl/server';
import {hasLocale} from 'next-intl';
import {routing} from './routing';

export default getRequestConfig(async ({requestLocale}) => {
  const requested = await requestLocale;
  const locale = hasLocale(routing.locales, requested)
    ? requested
    : routing.defaultLocale;

  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL locale segment → middleware | Locale value from URL path passes through middleware into app |
| messages/*.json → component render | Translation strings flow into JSX render output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | URL locale segment → middleware | mitigate | `hasLocale(routing.locales, locale)` in `request.ts` rejects any locale not in `['en', 'ar']`; `notFound()` called for invalid locales in layout.tsx (Plan 03) |
| T-02-02 | Tampering | messages JSON files | accept | Translation files are static build artifacts — no user input enters them; risk is negligible |
| T-02-03 | Information Disclosure | TypeScript type exports | accept | Types are compile-time only; no runtime secrets; no PII in type definitions |
</threat_model>
