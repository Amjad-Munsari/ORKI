## Typography

**Font pairing rationale:** Space Grotesk provides the geometric editorial character matching the dark underground streetwear aesthetic. IBM Plex Arabic mirrors Space Grotesk's geometric structure in the Arabic script. Both loaded via `next/font` for automatic FOUT prevention. Source: CLAUDE.md, STACK.md, PITFALLS.md P-RTL-3.

**Font switching via CSS `:lang()` selector — no component-level font logic:**

```css
:lang(en) { font-family: var(--font-space-grotesk), sans-serif; }
:lang(ar) { font-family: var(--font-ibm-plex-arabic), sans-serif; }
```

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Nav link labels, footer link labels, all body copy |
| Label | 13px | 400 (regular) | 1.4 | Language switcher toggle, category tags, utility labels |
| Heading | 20px | 600 (semibold) | 1.2 | Nav logo wordmark, section headers |
| Display | 28px | 600 (semibold) | 1.1 | Page-level headings (not used in Phase 1 nav/footer — reserved for Phase 4 home hero) |

**Arabic body size adjustment:** Arabic body text renders at 17px (1px bump from 16px) to maintain visual parity with Latin body at the same rendered optical size. This is a CSS-level override: `:lang(ar) { font-size: 1.0625rem; }`.

**Letter spacing:**
- Heading and Display roles: `letter-spacing: -0.02em` (tight, editorial)
- Body and Label roles: `letter-spacing: 0` (default, legible)

**Weights in use (exactly 2):** 400 (regular) and 600 (semibold). No other weights.

## Color

**Palette constraint from CLAUDE.md:** BLACK AND WHITE ONLY — no other palette. Dark-first.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (pure black) | Page background, nav background, footer background |
| Secondary (30%) | `#111111` (near-black) | Nav hover surface, footer surface, card backgrounds |
| Accent (10%) | `#FFFFFF` (pure white) | Text on dark surfaces, borders, interactive element fill |
| Destructive | n/a | No destructive actions exist in Phase 1 |

**Accent reserved for (explicit list):**
1. Primary text rendered on dark backgrounds (nav labels, footer links)
2. The language switcher active-state indicator (underline or background highlight)
3. Logo wordmark / brand name "ORKI"
4. Interactive border on nav hover states (1px white border)
5. Placeholder image inner logo mark on dark-field placeholder

**Accent is NOT used for:** general decorative elements, background fills, gradients.

**Border treatment:** `1px solid rgba(255, 255, 255, 0.12)` — low-opacity white creates subtle structural separation on dark surfaces without breaking the monochrome palette.

**Placeholder image color treatment:** Dark-field editorial — background `#0A0A0A` (near-black, not pure black, for depth), with a centered ORKI logo mark in `rgba(255, 255, 255, 0.15)` (ghost white, intentionally muted). Never a grey box, never a spinner. Source: FOUND-04, CLAUDE.md, PITFALLS.md P-FASHION-3.

## RTL Interaction Contract

Phase 1 establishes the RTL architecture that every subsequent phase inherits. This section defines the non-negotiable interaction rules.

**Language switching interaction:**
- Language toggle is a link: clicking it navigates to the same route in the opposite locale (`/en/...` ↔ `/ar/...`)
- Both `lang` and `dir` attributes update atomically on `<html>` — never one without the other
- Transition: no animation on the layout direction switch itself (direction change is instant to avoid disorienting motion)

**Navigation direction:**
- Desktop nav: links flow from inline-start. In EN: left-to-right. In AR: right-to-left. Use `display: flex` — direction inherits from `dir` automatically.
- Mobile nav drawer: slides in from `inline-end` in both locales (from right in EN, from left in AR). Uses CSS logical property `inset-inline-end: 0`.
- Hamburger icon: standard three-line icon. Morphs to X on open using Motion `AnimatePresence` (200ms ease-out, `scale(0.95) opacity(0)` exit). Not directional — does not flip in RTL.

**Directional icons in Phase 1:**
- Chevron in mobile nav items (if used for sub-navigation): wraps in `<DirectionalIcon>` — `scaleX(-1)` applied via `[dir="rtl"]` CSS. In Phase 1, navigation is flat (no chevrons needed unless mobile sub-items are added).
- External link icons: not directional, no flip.

**CSS logical properties enforcement (Phase 1 must establish this for the project):**
- ESLint rule: add `eslint-plugin-tailwindcss` with custom rule blocking `ml-*`, `mr-*`, `pl-*`, `pr-*`, `left-*`, `right-*`, `border-l-*`, `border-r-*` Tailwind utilities.
- All spacing must use: `ms-*`, `me-*`, `ps-*`, `pe-*`, `start-*`, `end-*`, `border-s-*`, `border-e-*`.
- This is enforced at the tooling level, not as a code review suggestion.

## Animation Contract

Phase 1 installs the animation library and establishes presets. No scroll animations exist yet (Phase 4). Only nav open/close and locale-aware motion apply.

**Library:** Motion (`motion/react`) — installed in Phase 1, used for all motion going forward. No mixing CSS keyframes and Motion. Source: PITFALLS.md P-FF-5.

**Animation presets (defined in `/src/lib/animation-presets.ts`):**

| Preset | Duration | Easing | Purpose |
|--------|----------|--------|---------|
| `nav-enter` | 200ms | `cubic-bezier(0.23, 1, 0.32, 1)` | Mobile nav drawer slide in |
| `nav-exit` | 160ms | `cubic-bezier(0.32, 0.72, 0, 1)` | Mobile nav drawer slide out |
| `fade-in` | 200ms | `cubic-bezier(0.23, 1, 0.32, 1)` | General element enter |
| `fade-out` | 150ms | `cubic-bezier(0.32, 0.72, 0, 1)` | General element exit |

**Direction-aware animation values:**
- Mobile nav drawer: enter from `translateX(100%)` in EN, `translateX(-100%)` in AR (inline-end edge)
- Derive direction offset via a `useDirection()` hook returning `1` (LTR) or `-1` (RTL), then multiply. Source: PITFALLS.md P-RTL-6.

**Reduced-motion rule (applies from Phase 1 onward):**
```tsx
const shouldReduceMotion = useReducedMotion(); // from motion/react
// Replace transform-based motion with opacity-only fade when true
```
All Motion variants must declare a `reducedMotion` variant that substitutes `opacity` for all `x`/`y`/`scale` transforms. Source: PITFALLS.md P-GEN-4.

**Button press feedback (all phases):** `scale(0.97)` on `:active`, `transition: transform 160ms ease-out`. Applied via Tailwind `active:scale-[0.97] transition-transform duration-[160ms]`. Source: emil-design-eng skill.

**Hover states:** Nav links use `opacity` change only (0.6 → 1.0, `transition: opacity 150ms ease`). No scale or translate on hover — high-frequency interactions must not animate. Source: emil-design-eng skill Animation Decision Framework.

## Image System Contract

**Aspect ratio lock (enforces PITFALLS.md P-PERF-2, P-FASHION-3):**

| Slot | Ratio | CSS | Container Pattern |
|------|-------|-----|------------------|
| Catalog card | 3:4 | `aspect-[3/4]` | `<div class="aspect-[3/4] relative overflow-hidden">` |
| PDP hero | 4:5 | `aspect-[4/5]` | `<div class="aspect-[4/5] relative overflow-hidden">` |
| Nav logo | fixed height | 32px height | Inline SVG or `next/image` with explicit height |

**Placeholder component specification (`<PlaceholderImage />`):**
- Background: `#0A0A0A`
- Logo mark: centered ORKI wordmark SVG at `rgba(255, 255, 255, 0.15)` opacity (ghost)
- No spinner, no grey box, no "loading" text
- Alt text received as prop (never empty)
- Receives `aspectRatio` prop: `"3/4"` | `"4/5"` — sets the `aspect-ratio` CSS
- When real image arrives: swap `src` prop only — no layout changes required

**`next/image` enforcement:**
- ESLint rule `@next/eslint-plugin-next/no-img-element` enabled — all images via `next/image`
- Above-fold images: `priority` prop required
- All `next/image` uses either explicit `width`/`height` OR `fill` prop inside positioned container
- Modern formats: configure `next.config.ts` with `formats: ['image/avif', 'image/webp']`

## Responsive Layout Contract

Breakpoints (from REQUIREMENTS.md FOUND-05):

| Breakpoint | Width | Nav behavior |
|-----------|-------|-------------|
| Mobile | 375px | Hamburger menu, full-screen overlay or Sheet drawer |
| Tablet | 768px | May collapse to hamburger or show partial nav — define during execution |
| Desktop | 1280px+ | Full horizontal navigation |

**Nav layout decision:** Desktop nav: horizontal flex. Mobile (< 768px): hamburger toggle with Sheet drawer sliding from inline-end.

**Content max-width:** `max-w-[1280px] mx-auto` — constrains content at large viewports. Full-bleed backgrounds (black) extend to viewport edge.

**Focal points:** Primary visual anchor of the nav bar is the ORKI logo wordmark (accent white `#FFFFFF`, Heading role 20px/600, positioned at inline-start). Secondary visual anchor is the language switcher (Label role 13px/400, positioned at inline-end). Footer primary anchor: brand copyright line (Label role, inline-start).

**No horizontal scroll at any breakpoint.** All widths responsive.

## Key Constraints Inherited by All Phases

These decisions made in Phase 1 are permanent constraints for the project:

1. **CSS logical properties only** — physical directional properties banned via ESLint from first commit
2. **`lang` + `dir` atomic update** — language switching must always update both attributes together
3. **Motion library installed in Phase 1** — no CSS keyframe animations for motion that Motion will own
4. **Space Grotesk + IBM Plex Arabic** — font pairing locked; no other typefaces
5. **Black and white palette only** — no other colors introduced in any phase
6. **`next/image` only** — raw `<img>` tags banned via ESLint
7. **Aspect ratio locked per slot** — catalog 3:4, PDP hero 4:5, set in Phase 1 placeholder component
8. **Western numerals in Arabic locale** — use `'ar-SA-u-nu-latn'` locale string for `Intl.NumberFormat`
9. **Animation only via `transform` and `opacity`** — no layout-triggering property animations
