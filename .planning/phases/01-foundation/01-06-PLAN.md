Replace `src/app/[locale]/page.tsx` with a placeholder home page that demonstrates both aspect ratios and provides the visual verification surface for Phase 1 sign-off.

The page is a Server Component. It uses `useTranslations` to render bilingual UI. It renders:
1. A 4:5 hero slot (full-width PDP-style placeholder, above fold)
2. A grid of 3:4 catalog card slots (below fold)

```typescript
import {useTranslations} from 'next-intl';
import {PlaceholderImage} from '@/components/PlaceholderImage';

export default function HomePage() {
  const t = useTranslations('Placeholder');

  return (
    <div className="w-full">

      {/* Above-fold: 4:5 hero slot — full bleed, max-height constrained on desktop */}
      <section className="w-full max-h-[80vh] overflow-hidden">
        <PlaceholderImage
          aspectRatio="4/5"
          alt={t('imageAlt')}
          priority
          className="w-full"
        />
      </section>

      {/* Below-fold: 3:4 catalog card grid */}
      <section className="max-w-[1280px] mx-auto px-6 py-12">
        <div className="grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4">
          {Array.from({length: 4}).map((_, i) => (
            <PlaceholderImage
              key={i}
              aspectRatio="3/4"
              alt={t('imageAlt')}
            />
          ))}
        </div>
      </section>

    </div>
  );
}
```

Key points:
- `priority` on the above-fold 4:5 slot — eager loading above fold (UI-SPEC requirement)
- `grid-cols-2 md:grid-cols-3 lg:grid-cols-4` — responsive grid without horizontal scroll at 375px
- `gap-4` is a neutral spacing utility (not directional) — safe for RTL
- `px-6` is physical padding (both sides equally) — correct for symmetric horizontal padding on a container
- No hardcoded copy — uses `t('imageAlt')` for alt text
