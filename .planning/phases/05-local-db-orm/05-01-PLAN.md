<read_first>

- .gitignore (verify postgres_data volume is not accidentally ignored)
</read_first>

<action>
Create `docker-compose.yml` in the repo root with the following exact content:

```yaml
# docker-compose.yml
# Local development database — mirrors Supabase Postgres 15 for frictionless migration.
# Run: docker compose up -d
# Stop: docker compose down (data persists in named volume)
# Destroy data: docker compose down -v

services:
  postgres:
    image: postgres:15-alpine
    container_name: orki_db
    environment:
      POSTGRES_USER: orki
      POSTGRES_PASSWORD: orki
      POSTGRES_DB: orki_dev
    ports:
      - "5432:5432"
    volumes:
      - orki_postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U orki -d orki_dev"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  orki_postgres_data:
```

Credentials: user=`orki`, password=`orki`, db=`orki_dev`, port=`5432`.
This gives DATABASE_URL=`postgresql://orki:orki@localhost:5432/orki_dev`.
</action>

<action>
Check if `zod` is in `package.json`. If not, run `npm install zod`.

Create `src/lib/env.ts` with the following exact content:

```typescript
/**
 * Environment variable validation using Zod.
 * This module validates all required env vars at import time.
 * Import this file in next.config.ts to fail at build time, not at runtime.
 *
 * Add new env vars here when new phases introduce them (Phase 6: auth vars, etc.)
 */
import { z } from 'zod';

const envSchema = z.object({
  DATABASE_URL: z.string().url({
    message: 'DATABASE_URL must be a valid PostgreSQL connection URL (e.g. postgresql://user:pass@host:5432/db)',
  }),
  NODE_ENV: z
    .enum(['development', 'production', 'test'])
    .default('development'),
});

const parsed = envSchema.safeParse(process.env);

if (!parsed.success) {
  console.error('❌ Invalid environment variables:');
  console.error(parsed.error.format());
  process.exit(1);
}

export const env = parsed.data;
```
</action>

<acceptance_criteria>
- `src/lib/env.ts` exists

- File contains `DATABASE_URL: z.string().url(`
- File contains `process.exit(1)`
- File exports `export const env = parsed.data`
- File does not import from any other src/ file (it must have zero internal dependencies to avoid circular imports)
</acceptance_criteria>
</task>
