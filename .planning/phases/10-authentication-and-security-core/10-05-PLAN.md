### Task 5.1: Create `src/lib/cart/merge-on-signin.ts` with race-safe merge

<intent>
The merge function is invoked from two places: (a) `getOrCreateCart()` (defensively, in case sign-in happens via a path that doesn't go through `signInAction`); (b) `signInAction` (proactive, immediately after Supabase auth succeeds). Both call sites get idempotency.

CONTEXT.md merge semantics: if user has no server cart → claim the guest cart (set its `userId`); if user already has a server cart → prefer it, silently DELETE the guest cart (no item-level union per CONTEXT). Wrap in transaction with `SELECT … FOR UPDATE` on the user's cart row to prevent two-tab race.
</intent>

<files>src/lib/cart/merge-on-signin.ts</files>

<approach>
Per 10-PATTERNS §"`src/lib/cart/merge-on-signin.ts`":

```ts
import 'server-only';
import { db } from '@/lib/db/client';
import { carts } from '@/lib/db/schema';
import { eq, and, sql, isNull } from 'drizzle-orm';

/**
 * One-shot guest-cart merge on first authenticated request.
 *
 * Semantics (CONTEXT.md §"Guest cart merge behavior"):
 *  - If user has no server cart → claim the guest cart by setting userId = userId.
 *  - If user already has a server cart → prefer it. DELETE the guest cart silently.
 *
 * Race-safety (RESEARCH §7 #9): wrapped in db.transaction with SELECT … FOR UPDATE on
 * the user's cart row. Two simultaneous tabs both calling this function for the same
 * user will see one of two states post-acquisition:
 *   1. No user cart yet → first tab claims the guest cart; second tab sees the guest
 *      cart already userId-bound (or absent) and does nothing.
 *   2. User cart present → both tabs delete the guest cart (idempotent: DELETE on a
 *      missing row is a no-op).
 *
 * Idempotent. Never throws raw — all errors are swallowed via the migrateLocalCartAction
 * precedent (src/app/actions/cart.ts:96-112).
 */
export async function mergeGuestCartIntoUserCart(
  userId: string,
  sessionId: string | null | undefined,
): Promise<void> {
  if (!sessionId) return;
  try {
    await db.transaction(async (tx) => {
      // Lock the user's existing cart row (if any) for the duration of the txn.
      const [userCart] = await tx
        .select()
        .from(carts)
        .where(eq(carts.userId, userId))
        .for('update');

      if (userCart) {
        // User already has a server cart → silently discard the guest cart.
        await tx
          .delete(carts)
          .where(and(eq(carts.sessionId, sessionId), isNull(carts.userId)));
        return;
      }

      // No user cart → claim the guest cart by assigning userId.
      // Only update IF the guest row is still ownerless (defends against another
      // tab having already claimed it).
      await tx
        .update(carts)
        .set({ userId })
        .where(and(eq(carts.sessionId, sessionId), isNull(carts.userId)));
    });
  } catch (err) {
    console.error('[mergeGuestCartIntoUserCart]', err);
    // Never throw — auth flow must not break on merge failure.
  }
}
```

The Drizzle `.for('update')` API may need adjustment based on the installed Drizzle version. If `.for('update')` is unavailable, use raw SQL via `sql\`SELECT … FROM carts WHERE … FOR UPDATE\`` per the Phase-8 transaction precedent in `src/lib/orders/server.ts`.
</approach>
