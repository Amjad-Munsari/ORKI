### Task 1.6: Replace `src/middleware.ts` with combined Supabase-refresh + next-intl middleware

<intent>
Critical ordering invariant (RESEARCH §7 #10): Supabase session refresh MUST run on every request BEFORE next-intl redirects, otherwise an expiring JWT can cause `getUser()` in the destination page to return null. Combine both into a single `middleware()` function.

`type=tdd` is NOT applied because middleware is infrastructure wiring; the combined-middleware behavior is exercised end-to-end by Plan 10-04's auth-page integration tests.
</intent>

<files>src/middleware.ts</files>

<approach>
- Per 10-PATTERNS §"`src/middleware.ts`":
- Replace the current 4-line file with the combined middleware verbatim from PATTERNS §"What to diverge":
  ```ts
  import { type NextRequest, NextResponse } from 'next/server';
  import createMiddleware from 'next-intl/middleware';
  import { createServerClient } from '@supabase/ssr';
  import { routing } from './i18n/routing';
  import { env } from '@/lib/env';

  const intlMiddleware = createMiddleware(routing);

  export async function middleware(request: NextRequest) {
    let response = intlMiddleware(request);

    const supabase = createServerClient(
      env.NEXT_PUBLIC_SUPABASE_URL,
      env.NEXT_PUBLIC_SUPABASE_ANON_KEY,
      {
        cookies: {
          getAll() { return request.cookies.getAll(); },
          setAll(cookiesToSet) {
            cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
            response = intlMiddleware(request);
            cookiesToSet.forEach(({ name, value, options }) =>
              response.cookies.set(name, value, options)
            );
          },
        },
      },
    );
    await supabase.auth.getUser();

    return response;
  ```
